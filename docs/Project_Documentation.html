Project Overview

  • Type: Full-stack e-commerce application
  • Backend: Django REST API (Django 6.x)
  • Frontend: React (Vite)
  • Database: SQLite (development)
  • Auth: JWT via djangorestframework-simplejwt
  • Payments: Stripe integration (webhook present)
  • Media: Uploaded product images stored under media/products/

Repository layout (important folders)

  • backend/ — Django project and apps (settings, manage.py, apps)
  • frontend/ — React + Vite app (src, package.json)
  • media/ — uploaded images (products sample images)
  • docs/ — documentation and conversion scripts

Key configuration notes

  • Backend settings: backend/backend/settings.py
    • AUTH_USER_MODEL = accounts.User
    • MEDIA_ROOT set to media/; MEDIA served in DEBUG via static()
    • REST framework config contains typos (verify keys such as DeFAULT_PAGINATION_CLASS)
    • Stripe keys present in settings — move to environment variables for production
  • Dependencies (backend): backend/requirements.txt
  • Frontend deps: frontend/package.json

How to run (development)

Backend (assumes Python virtualenv activated):

pip install -r backend/requirements.txt
cd backend
python manage.py migrate
python manage.py runserver

Frontend:

cd frontend
npm install
npm run dev

Main Django apps & important models

  • accounts — Register, login, profile, password reset (see accounts/urls.py)
  • products — Category, SubCategory, Product (see products/models.py)
  • orders — Cart, Order models and Stripe webhook (see orders/models.py)

Frontend overview

  • Routes and pages defined in frontend/src/App.jsx
  • Topbar implements search suggestions, cart badge, profile dropdown
  • main.jsx wraps App with CartProvider and BrowserRouter

API endpoints (high level)

  • Base prefix: /api/
  • Accounts/auth: /api/register/, /api/login/, /api/profile/, password reset endpoints
  • Stripe webhook: /stripe-webhook/

Conversion to PDF

This folder includes two conversion helpers:

  • docs/convert_html_to_pdf.bat — tries wkhtmltopdf, falls back to a Node Puppeteer script.
  • docs/convert_with_puppeteer.js — Node script using Puppeteer to convert the HTML to PDF (downloads Chromium when first run).
